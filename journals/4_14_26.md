### **NOTES**
Previously for prompt/settings windows I would creat a new display on top of the existing render in tk. Xwayland does not know how to handle the windows this method like Windos OS does. The solution was to have tk redraw the same window instead of creating a new one. This also solves a bug where additional windows would stay on top when the application was minimized (not a problem in kiosk environment).
